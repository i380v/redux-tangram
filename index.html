redux-tangram

redux-tangram 是一个构建于Redux和Reudx-Saga之上的 javascript 库， 能够简化 Redux 调用过程，减少 Redux 使用过程中的模版代码。

使用 redux-tangram 使 Redux 中 action,reducer,selector 等概念根据具体的功能模块更内聚，并可随意组合

redux-tangram 提供了完善的类型支持，如果你使用 typescript，redux-tangram 会根据你模块的定义对象进行推导，生成的 redux-tangram module 会获得完善的类型提醒

Installation

npm install --save redux-tangram

Get Start

以 todos 为例，我们先以一个 js 对象定义好 todos 模块的数据格式和包含的方法，调用createModule方法生成对应的模块

todos.js

import { createModule } from "redux-tangram";

export default createModule({
  state: [],
  mutations: {
    add(state, todo) {
      return [...state, todo];
    }
  },
  sagas: {
    *remoteAdd() {
      const remoteTodo = yield /* 一些异步的或包含副作用的调用 */ fetchNew();
      this.add(remoteTodo);
    }
  }
});

然后我们调用tangramFactory生成一个 redux 支持格式的增强因子（enhancer），该增强因子也包含toStore方法支持以配置的形式生成 redux store

index.js

import tangramFactory from "redux-tangram";
import extraReducer from "./extraReducer";
import middlewares from "./middleware";
import enhancers from "./enhancer";

const store = tangramFactory().toStore({
  reducer: extraReducer,
  middleware: middlewares,
  enhancer: enhancers
});

或者你也可以用tangramFactory替换redux的compose方法组合其他的 enhancer，middleware 来生成 store

import { createStore, compose, applyMiddleware } from "redux";

const store = createStore(
  extraReducer,
  tangramFactory(...enhancers, applyMiddleware(...middlewares))
);

如果没有其他 enhancer，middleware 请调用tangramFactory()。该方法会生成内置的 enhancer 改变我们的 store

以上两种方式生成的 store 都会包含root属性，这是一个空的 module，作为我们整个 module tree 的根节点，其他模块可以以树形结构添加到root模块上。每个模块本身是一个函数，我们可以按(key:string, module: Module | ModuleDefination)=>Module的格式调用它，如下

import todos from "./todos";

store.root("todos", todos);

这样 todos 模块被添加到 root 的'todos'属性节点下，模块在整个 module tree 上的位置和该模块 state 在 state tree 上的位置一一对应。因此，完成以上后我们的 store state tree 大概是以下的格式

{
  "todos": []
}

之后你就可以按函数的形式调用 module 中暴露的 act 方法了，并且 module 本身在单参数并参数类型为对象时按照 module 层次结构返回对应 state，以 React Component 为例

TodoScreen.js

import React from "react";
import { connect } from "react-redux";
import todos from "./todos";

class TodoScreen extends React.Component {
  render() {
    // React render
  }

  addTodo() {
    //暴露的action create已经绑定到store的dispatch，所以直接调用
    todos.add("job 1");
  }

  fetchTodo() {
    todos.remoteAdd();
  }
}

export default connect(ignoreWholeState => {
  //无参调用时，返回module对应state，此处可以忽略全局state，只当作redux的订阅方法使用
  //并且此处可以给你该state的静态类型提醒
  const todosState = todos();
  return {
    todos: todosState
  };
})(TodoScreen);

以上我们可以直观的得到如下好处

  • 直观的定义你使用 redux 时需要的一切，相关联的功能按照 module 以树形组合并划分，再也没有零散的 action creater， reducer， dispatch
  • redux-tangram 为你隐去了 action，一般的 redux 模版中，action 作为描述事件的对象，它的格式耦合了对应的 action creator 和 reducer。redux-tangram 背后虽然还是以dispatch(action)的方式执行，但会为你自动生成 action，action 中的载荷（payload）通过函数的参数来传递。这大大简化了其中的模版代码
  • 每个 module 都是高内聚的，调用的地方需要显示import 'your-module'。当定义有变化时，借助编译器我们能马上发现调用错误的地方
  • module 可以灵活的相互组合，借助 module 的 selector 方法，我们其实都不需要关心 module 在 store state tree 中的位置，比如当我们修改 todos 的位置
store.root("td", todos);

我们甚至不需要修改其他代码，而如果是一般的 redux 模版，TodoScreen.js中的代码可能是这样的

TodoScreen.js

export default connect(state => {
  const todosState = state["todos"];
  return {
    todos: todosState
  };
})(TodoScreen);

类似的 selector 调用都需要随 state 层次结构变化而改变
