Module

Module

module是一个包含MutationAct，SagaAct和其他次级moudle的对象，也是一个由State的 selecotr 函数，属性选择函数等重载（overloads）的函数

Definition

module通过向createModule函数传入定义对象生成。一个定义对象内可以选择包含以下属性

  • state: any - state 属性定义了该module初始的 state 值，该 state 值会是mutation函数的第一个参数。当该module拥有子module时，state 必须是普通对象，因为所有子 module 的 state 会以它们在该 module 中的位置被添加到该 state 中

  • mutations: {[key: string]: Mutation} - mutations 对象内的属性是Mutation定义函数，它们将会转化成MutationAct并以当前属性名赋值到最终生成的module中

  • sagas: {[key: string]: Saga} - sagas 对象内的属性值是Saga函数，它们将转化为SagaAct并以当前属性名赋值到最终生成的module中

  • modules - modules 属性值包含以下两种格式

    • {[key: string: Module | ModuleDefinition]} - 一个对象，属性值是子moudle或该module的定义对象，子module会以在该对象中的位置被添加到其父module

    • (key:string) => Module | ModuleDefinition | void - 函数，用于动态 module 形式，入参 key 是子 module 的属性名，返回module对象或module定义对象会在该属性名下生成子module

Function Overloads

module作为函数可以按以下方式调用

  • (key: string, subModule: Module | ModuleDefinition) => Module - 当第一个入参是字符串，第二个入参可以是一个module对象或module的定义对象。这么调用会将该module添加为父module的对应 key 的下层，返回被添加的module

  • (key: string)=> MutationAct | SagaAct | Module | undefined - 当只有一个参数并且是字符串时，会返回该module属性，当不存在该属性时默认返回undefined，当该moudle存在动态子module时有所不同，见下条

  • (key: string)=> module - 当module定义对象存在modules属性并且该属性是函数时，表示该module存在动态次级module。此时当以唯一的字符串调用moudle函数并且不满足上一条件时，redux-tangram会调用modules函数，并传递该 key 字符串，当函数返回moudle时，会将该module注册为调用module在该 key 上的子module并返回该module。当modules函数返回的是module定义对象时，先生成对应module再执行。其他的返回对象视作无效

  • ()=> State - 无参调用返回当前State

Dynamic Module

当一个 module 的功能可以按常量切分成独立的块并且需要在运行时确定区分的常量，我们可以应用动态模块的功能

const mainModule = createModule({
  modules: key => {
    return {
      state: [],
      mutations: {
        add(state, some) {
            return [...state, some]
        }
      }
    };
  }
});

在模块定义对象中指定modules为函数后，可以对mainModule按如下链式调用

mainModule('sub0').add('one')

其中包含的步骤为

  1. const subMoudle = mainModule('sub0') - 取mainModule下名为sub0的子module，如果不存在则调用modules定义函数生成

  2. subModule.add('one') - 调用sub0子module的MutationAct

Typing

当我们如下调用createModule函数

const mockModule = createModule({
  state: {
    count: 1
  },
  mutations: {
    add(state, num: number) {
      return {
        ...state,
        count: state.count + num
      };
    }
  },
  sagas: {
    *asyncAdd(num: number) {
      yield delay(1000);
      yield call(this.add, num);
    }
  },
  modules: {
    list: {
      state: [] as string[],
      mutations: {
        append(state, batchList: string[]) {
          return [...state, ...batchList];
        }
      }
    }
  }
});

可以得到类似如下类型结构的module

type MockModule = {
  //MutationAct
  add(num: number): { type: string; payload: [number] };
  //SagaAct
  asyncAdd(num: number): Task;

  //Module Function Overloads
  (key: string, subModule: Module | ModuleDefinition): Module;
  (key: string): Module | MutationAct | SagaAct | undefined;
  (): { count: number };

  //SubModule
  list: {
    append(batchList: string[]): { type: string; payload: [string[]] };

    (key: string, subModule: Module | ModuleDefinition): Module;
    (key: string): Module | MutationAct | SagaAct | undefined;
    (): string[];
  };
};
