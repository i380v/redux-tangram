Mutation

Mutation

mutation是redux-tangram中修改 Sate 唯一的入口

Definition

mutation的定义函数类似reducer(一个 module 中的 mutation 事实上会组合成一个reducer)，接受第一个参数为当前 module 的 state，返回新的 State 对象会替换掉 Store 中 State Tree 对应部分，完成 State 的更新

const countModule = createModule({
  state: {
    count: 0
  },
  mutations: {
    add(state, num) {
      return {
        count: state.count + num
      };
    }
  }
});

要执行Mutation，我们需要调用 module 中对应的 MutationAct，并传入需要的参数

countModule.add(10);

MutationAct

createModule会遍历mutations定义对象，每个mutation会在module中生成相同属性名的可执行函数我们称为MutationAct

MutationAct相当于mutation的代理，我们不该直接调用mutation函数，而是调用MutationAct来让redux-tangram触发mutation

Principle

mutation遵循和reducer一样的原则。它们都是纯函数，不在其中执行有副作用的操作，并且只返回新的 State 对象而不是修改前一个 State
