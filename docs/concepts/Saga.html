Saga

Saga

Saga是redux-tangram中执行副作用操作的地方。Redux-Saga为我们提供了Saga的概念和功能

Definition

Saga是一个 ES6 的 Generator 函数，如果你并不熟悉 Generator，可以在这里找到一些资料

import {} from "redux-tangram/effects";

const fetchModule = createModule({
  state: [],
  mutations: {
    addData(state, data) {
      return [...state, data];
    }
  },
  sagas: {
    *fetchData(id) {
      const data = yield call(Api.fetchData, id);
      this.addData(data);
    }
  }
});

要执行Saga，我们需要调用 module 中对应的 SagaAct，并传入需要的参数

fetchModule.fetchData(1);

SagaAct

createModule会遍历 sagas 定义对象，每个saga会在 module 中生成相同属性名的可执行函数我们称为SagaAct

要触发Saga我们需要调用SagaAct并传入定义的参数

Task

调用SagaAct返回Task类型的结果，它反应了该次 Saga 的执行过程

  • task.isRunning() - 如果 Saga 还在执行中返回 true

  • task.isCancelled() - Saga 被取消时返回 true

  • task.result() - 返回 Saga 结果，未完成时返回undefined

  • task.error() - Saga 执行异常时返回该异常

  • task.toPromise() - 返回一个 Promise resolve 返回 Saga 执行结果，reject 返回异常

  • task.cancel() - 如果该 Task 还在执行中，中断执行

我们为返回的 Task 增加了then和catch方法，使它成为类 Promise 对象，我们可以直接将它当作 Promise 处理

fetchModule
  .fetchData(1)
  .then(data => {
    console.log("resolve data" + data);
  })
  .catch(error => {
    console.error(error);
  });

Context

Saga执行时的 Context 是它所在 module，我们可以直接应用this.add(data)方式调用该 module 其他的SagaAct或MutationAct
