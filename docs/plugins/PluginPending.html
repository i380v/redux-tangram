Plugin Pending

@redux-tangram/plugin-pending

plugin-pending 插件自动为你记录每个 sagaAct 的执行状态

Install

npm install --save @redux-tangram/plugin-pending

Usage

首先在 tangramFactory 方法的 options 中添加 pendingPlugin

configStore.js
import tangramFactory from "redux-tangram";
import createPendingPlugin from "@redux-tangram/plugin-pending";

//导出pending plugin对象在需要的地方使用isPending方法
export const pending = createPendingPlugin();

export const store = tangramFactory({
  plugins: [pending]
}).toStore();

@redux-tangram/plugin-pending 导出的是pluginCreator，执行生成plugin instance

之后在需要的地方引用以上 pending 对象，其中包含 isPending 方法可以查询 module 或 sagaAct 是否在执行状态

import { createModule } from "redux-tangram";
import { delay } from "redux-tangram/effects";
import { store, pending } from "./configStore";
const mockModule = createModule({
  sagas: {
    *fetchData() {
      yield delay(1000); //mock async fetch
    },
    *asyncOperate() {
      yield delay(1000); //mock async operate
    }
  }
});
store.root("mock", mockModule);
mockModule.fetchData();

const isFetchDataPending = pending.isPending(mockModule.fetchData); //ture
const isAsyncOperatePending = pending.isPending(mockModule.asyncOperate); //false
const isModulePending = pending.isPending(mockModule); //true

Pending State

sagaAct的执行状态为其被调用后的所有Task执行状态的总和，当还有Task在执行状态时，该sagaAct就处于执行状态

module的执行状态为其下所有sagaAct的执行状态和子module(当relayOnSubModule)为true)的执行状态的总和，当它们其中一个处于执行状态时，该module就还在执行状态

Pending Module

pendingPlugin 在内部维持一个redux-tangram格式的module，并通过该module记录各个sagaAct的执行状态。该module会在store生成时被添加到module tree的第一层，默认名为pending

Api

createPendingPlugin(options)

@redux-tangram/plugin-pending默认导出createPendingPlugin，执行生成pendingPlugin对象

  • options: Object - plugin配置对象

    • name: string - 内部生成的pengingModule在store中应用的模块名，默认为pending

    • relayOnSubModule: boolean - 模块的执行状态是否受其子模块影响，默认为true，子模块为执行状态时其父模块也为执行状态

    • configName: string - 控制pendingPlugin开关的属性名

pendingPlugin

createPendingPlugin)函数生成的plugin对象，应用于tangramFactory的plugins option，如果你的程序中有多个的store，请为它们应用独立的pendingPlugin

  • pendingPlugin.isPending(element) - 返回该组件是否在执行状态

    • element: SagaAct - 返回该SagaAct的执行状态

    • element: Module - 返回该Module的执行状态
