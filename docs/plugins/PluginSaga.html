Plugin Saga

@redux-tangram/plugin-saga

plugin-saga 为你提供基于配置的增强 SagaAct 的方式，主要包括Redux-Saga中的 throttle, debounce 等 effects

Install

npm install --save @redux-tangram/plugin-saga

Usage

首先在 tangramFactory 方法的 options 中添加 sagaPlugin

import tangramFactory from "redux-tangram";
import createSagaPlugin from "@redux-tangram/plugin-saga";

const sagaPlugin = createSagaPlugin();

const store = tangramFactory({
  plugins: [sagaPlugin]
}).toStore();

然后在createModule时可以添加上配置属性

import { createModule } from "redux-tangram";
import createSagaPlugin from "@redux-tangram/plugin-saga";

const mockModule = createModule({
  sagas: {
    *fetchData() {
      // implementation
    },
    *asyncOperate() {
      // implementation
    }
  },
  // 在这个属性下为对应名称的SagaAct配置
  $saga: {
    fetchData: createSagaPlugin.takeLeading(),
    asyncOperate: "debounce"
  }
});

SagaEnhancer

type SagaEnhancer = <F extends () => IterableIterator<any>>(
  saga: F
) => (...args: Parameters<F>) => IterableIterator<any>;

SagaPlugin 通过应用 SagaEnhancer 到对应 Saga 来增强 Saga 的行为。

Config

$saga属性可以以多种形式配置 SagaEnhancer

  • {[SagaActName: string]: any} - 对象中的属性名对应SagaAct的名称，其值作为该SagaAct的配置

  • (SagaActName: string) => any | void - 为每个SagaAct返回对应的配置

每个SagaAct对应配置可以是三种格式

  • SagaEnhancer - 该enhancer会直接作用于Saga，并替换

  • string - 内置的SagaEnhancer可以通过字符串形式引入。debounce(400)，有参数时直接以表达式传入参数

  • Array<SagaEnhancer | string> - 为数组时，按顺序执行Enhancer

Api

createSagaPlugin(options)

生成一个 sagaPlugin

  • options: Object - plugin 配置对象

  • configName: string - 控制 sagaPlugin 配置项的属性名，默认$saga

内置的 SagaEnhancer

  • createSagaPlugin.takeLast() - 只执行最后一个Task，会取消之前所有未执行完的Task

  • createSagaPlugin.takeLeading() - 阻止并直接取消之后的Task直到前一个Task执行完毕

  • createSagaPlugin.throttle(ms?: number) - 每隔ms时间才重新执行Task，之间生成的Task最后一个会延迟到ms间隔结束后执行，其他的直接取消

    • ms: number - 间隔毫秒数，默认500

  • createSagaPlugin.debounce(ms?: number) - 当一个新Task开始时延迟ms时间才执行，延迟期间如果有新的Task开始，该延迟的Task会被取消，新的Task重复上述过程

    • ms: number - 间隔毫秒数，默认500

使用字符串takeLast，takeLeading，throttle，debounce应用以上内置SagaEnhancer
